# 디스코드 채팅 미러 봇 설정 가이드

경매 방송 화면의 채팅 패널에 **지정한 디스코드 채널의 실제 메시지**를 실시간으로
보여주기 위한 봇 설정 방법입니다. 단방향(디스코드 → 웹) 읽기 전용입니다.

> 봇을 설정하지 않아도 앱은 동작합니다. 이 경우 채팅 패널은 시뮬레이션 메시지로
> 폴백됩니다. 실제 미러를 쓰려면 아래 1~6을 따라 `.env`를 채우세요.

## 1. 애플리케이션 & 봇 생성

1. https://discord.com/developers/applications 접속 → **New Application** 생성.
2. 좌측 메뉴 **Bot** → **Add Bot** (또는 기본 생성된 봇 사용).

## 2. 봇 토큰 발급

1. **Bot** 탭에서 **Reset Token** → 표시되는 토큰 복사.
2. 이 토큰은 비밀입니다. 깃에 커밋하지 말고 `.env`에만 넣으세요.

## 3. MESSAGE CONTENT Intent 활성화 (필수)

1. **Bot** 탭 → **Privileged Gateway Intents** 섹션.
2. **MESSAGE CONTENT INTENT** 를 **켭니다**.
   - 이 인텐트가 없으면 봇이 메시지 본문을 읽지 못해 채팅이 표시되지 않습니다.

## 4. 서버에 봇 초대

1. 좌측 메뉴 **OAuth2** → **URL Generator**.
2. **Scopes**: `bot` 체크.
3. **Bot Permissions**: `View Channels`, `Read Message History` 체크.
4. 생성된 URL을 브라우저에서 열어 대상 서버에 봇을 초대합니다.

## 5. 대상 채널 ID 복사

1. 디스코드 앱 → **사용자 설정 → 고급 → 개발자 모드** 켜기.
2. 미러링할 채널을 **우클릭 → 채널 ID 복사**.

## 6. 환경변수 설정

`.env` 파일에 다음을 채웁니다 (`.env.example` 참고).

```env
DISCORD_BOT_TOKEN="<2단계에서 복사한 토큰>"
DISCORD_CHANNEL_ID="<5단계에서 복사한 채널 ID>"
```

앱을 재시작하면, 해당 채널의 새 메시지가 경매 화면 채팅 패널에 실시간으로 나타납니다.

## 동작 메모

- 봇 메시지와 본문이 빈 메시지(첨부/임베드 전용)는 표시하지 않습니다.
- 발신자 색상은 디스코드 역할 색이 있으면 그 색을, 없으면 닉네임 기반 고정 색을 씁니다.
- 메시지는 서버 메모리에 최근 30개만 보관합니다(영구 저장 없음). 단일 프로세스 기준입니다.
- 로그인 실패 시 자동으로 시뮬레이션 폴백으로 강등되어 화면이 비지 않습니다.
